# MVP 任务拆分

> ⚠️ **本文档是原始设计规划文档**，创建于项目启动阶段。任务状态标记保留原样（均为 pending），不做更新。
> 实际完成情况请参阅 [project-status.md](project-status.md)。核心 Pipeline 已全部实现，当前瓶颈在 TTS 性能和口型匹配。
>
> **MVP 目标**: 输入任意视频，输出中文配音版本（不含口型修正）。
> **MVP 流程**: 视频解复用 → 语音识别 → 文本翻译 → 语音合成 → 音视频混流

---

**验收标准**:

- [ ] 依赖声明文件就绪，`pip install` 可一键安装
- [ ] 有 CLI 入口文件，可通过命令行调用
- [ ] `.gitignore` 已配置，排除临时文件和模型文件
- [ ] 有集中管理的配置文件

**状态**: ⬜ pending

### T-002: 核心依赖可用

**目标**: 确保所有 MVP 阶段需要的运行环境就绪。

**验收标准**:
- [ ] 视频/音频处理库可正常调用
- [ ] 语音识别模型可加载并识别音频
- [ ] 翻译服务可用（API 连通或离线引擎正常）
- [ ] 语音合成引擎可生成中文音频
- [ ] 系统已安装 FFmpeg 且可在命令行调用

## Step 1: 视频解复用

### T-003: 视频音频分离

**目标**: 从输入视频中分离出无声视频轨和音频轨。

**验收标准**:
- [ ] 支持常见视频格式（mp4、mkv、mov、webm 等）
- [ ] 输出无声视频轨，视频编码与原视频一致（不重新编码）
- [ ] 输出音频文件，格式为 16kHz 单声道 WAV
- [ ] 同时输出视频元信息（FPS、分辨率、编码格式、总时长、音频采样率）
- [ ] 处理失败时有明确错误信息（不支持的格式、文件损坏等）

**状态**: ⬜ pending

---

### T-004: 工作目录管理

**目标**: 每次运行在隔离的工作目录中产出中间文件，互不干扰。

**验收标准**:
- [ ] 以输入视频名称为标识创建工作目录
- [ ] 目录内按处理阶段划分子目录
- [ ] 同名视频复用同一工作目录，不重复创建
- [ ] 工作目录与源码目录隔离

**状态**: ⬜ pending

---

## Step 2: 语音识别

### T-005: 音频转带时间戳的文本

**目标**: 将音频中的语音转为文字，并标注每段对话的起止时间。

**验收标准**:
- [ ] 输入为 16kHz WAV 音频，输出为带有 `start/end/text` 字段的结构化数据
- [ ] 时间戳精度达到秒级（误差 < 0.5 秒）
- [ ] 自动检测源语言并标注
- [ ] 按自然句子切分段落（非逐词切分）
- [ ] 静音段被自动跳过，不在结果中出现
- [ ] 支持英语、日语、韩语等常见外语

**状态**: ⬜ pending

---

### T-006: 识别结果后处理

**目标**: 清理和规整 ASR 输出的文本段落，提升后续翻译和合成质量。

**验收标准**:
- [ ] 间隔过短（< 0.3 秒）的相邻段落自动合并
- [ ] 空白片段、纯标点片段被剔除
- [ ] 单段超过 15 秒的段落被二次拆分
- [ ] 每段文本非空
- [ ] 处理后不改变原有时间戳的起止边界

**状态**: ⬜ pending

---

## Step 3: 翻译

### T-007: 字幕文本翻译

**目标**: 将源语言字幕翻译为简体中文，保持原有时间信息。

**验收标准**:
- [ ] 每段文本完整翻译为简体中文
- [ ] 保留原时间戳信息（start/end 不变）
- [ ] 翻译结果口语化，适合朗读配音
- [ ] 翻译长度不过度膨胀（控制在原文 1.5 倍字符数以内）
- [ ] 翻译失败时有降级策略（如单句重试、切分重试）

**状态**: ⬜ pending

---

## Step 4: TTS 语音合成

### T-008: 中文文本转语音

**目标**: 将翻译后的中文文本逐段合成为语音。

**验收标准**:
- [ ] 每段中文文本生成对应的 WAV 音频文件
- [ ] 中文发音自然、清晰、可懂
- [ ] 支持选择不同发音人（至少 1 男 1 女）
- [ ] 每段生成耗时不超过 3 秒
- [ ] 合成失败时有重试机制

**状态**: ⬜ pending

---

### T-009: 音频拼接与格式统一

**目标**: 将所有语音片段拼接为一段完整的音频，格式与原始音轨一致。

**验收标准**:
- [ ] 所有片段按时间顺序拼接为单个 WAV 文件
- [ ] 采样率与原始音频一致（16kHz）
- [ ] 声道数与原始音频一致（单声道）
- [ ] 段落间有自然的短停顿（约 200ms 静音间隔）
- [ ] 存在一个总时长记录，便于后续与视频对齐

**状态**: ⬜ pending

---

## Step 5: 音视频合成

### T-010: 音视频混流

**目标**: 将新音轨替换到原视频中，输出最终合成视频。

**验收标准**:
- [ ] 视频画质与原始一致（视频轨不重新编码）
- [ ] 新音轨与视频轨正确同步
- [ ] 音视频长度不一致时可正确处理：（音频短则补静音，音频长则截断）
- [ ] 输出格式为 mp4，可在主流播放器正常播放
- [ ] 混流速度接近磁盘 I/O 速度（不引入额外计算瓶颈）

**状态**: ⬜ pending

---

### T-011: 音量平衡

**目标**: 输出音频的音量水平与原始视频一致，不过响也不过轻。

**验收标准**:
- [ ] 合成音频的整体音量与原始音频偏差不超过 ±3dB
- [ ] 无明显爆音或削波失真
- [ ] 不同片段之间音量过渡平滑，无突兀跳跃

**状态**: ⬜ pending

---

## 流程编排

### T-012: 流程编排器

**目标**: 串联全部处理步骤，形成一个端到端的自动化流程。

**验收标准**:
- [ ] 调用 single API / 函数即可跑完全流程
- [ ] 支持断点续跑：某步骤失败或中断后，重新运行会从上次成功的步骤继续
- [ ] 每步执行时输出可读的进度和耗时信息
- [ ] 步骤异常时给出明确的错误提示（哪步、什么原因）
- [ ] 各步骤的中间产物持久化保存在工作目录中

**状态**: ⬜ pending
